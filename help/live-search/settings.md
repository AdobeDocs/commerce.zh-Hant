# 設定

使用&#x200B;**設定**&#x200B;工作區來設定語意搜尋、價格Facet範圍和間隔，以及索引的預設語言。

![設定](assets/settings.png)

## 語意搜尋 {#semantic-search}

>[!AVAILABILITY]
>
>語意搜尋適用於使用Adobe Commerce 2.4.4版及更新版本的商家。

語意搜尋使用AI根據含義和內容來比對產品，而不只是精確的關鍵字。 啟用&#x200B;**[!UICONTROL Semantic search]**&#x200B;後，使用自然語言或文字與目錄不符的購物者仍可找到相關產品。 [!DNL Live Search]在店面的一個整合式搜尋體驗中提供關鍵字和語意比對。 語意搜尋可與您的現有設定搭配使用；[搜尋規則](rules.md)、[同義字](synonyms.md)、[面向](facets.md)、提升和[類別銷售](category-merch.md)會繼續套用。

**若要啟用語意搜尋（僅限PaaS）：**

1. 在Admin中，移至&#x200B;**行銷** > *SEO與搜尋* > **[!DNL Live Search]**。
1. 在&#x200B;**設定**&#x200B;工作區上，啟用&#x200B;**[!UICONTROL Semantic search]**。

   啟用後，搜尋會根據含義和內容比對產品，產生更相關的結果、較少的零結果搜尋，並改善轉換。

1. 按一下&#x200B;**[!UICONTROL Save]**。

   索引完成之後會更新搜尋結果。 對於中型型錄，編制索引最多可能需要半小時。 若為包含數百萬項產品的大型目錄，則可能需要數小時的時間。

>[!NOTE]
>
> 語意搜尋僅適用於&#x200B;**英文**&#x200B;目錄。 如果您將&#x200B;**語言**&#x200B;設定為非英文目錄（請參閱[語言](#language)），**[!UICONTROL Semantic search]**&#x200B;會自動停用。

如需優點、驗證指南、最佳實務、疑難排解及限制，請參閱[語意搜尋](semantic-search.md)。

### 欄位說明

| 欄位 | 說明 |
| --- | --- |
| 語意搜尋 | 啟用時，[!DNL Live Search]會使用關鍵字比對的意義與內容。 預先定義的目錄屬性會自動使用；不需要在「管理員」中進行屬性設定。 預設為[!DNL Adobe Commerce as a Cloud Service]啟用；PaaS商家會手動啟用。 |

## 價格多面向 {#price-faceting}

您可以指定價格範圍群組的數量，以及價格值在群組之間的分配方式。 每個價格範圍會依序與上一個群組重疊。 例如，間隔為20的五個群組會建立下列價格範圍：0-20、20-40、40-60、60-80和>80。 如果目錄中沒有足夠產品可填滿所有已定義的範圍，則會相應地調整可用群組的顯示。 例如：0-20、60-80、>80。

**若要設定價格刻面：**

1. 在Admin中，移至&#x200B;**行銷** > *SEO與搜尋* > **[!DNL Live Search]**。
1. 在&#x200B;*價格面向*&#x200B;下的&#x200B;**設定**&#x200B;工作區上，執行下列動作：
   * 輸入可用的&#x200B;**選擇數目**&#x200B;或價格群組。 透過[!DNL Live Search] 4.4.0，您可以定義最多100個價格群組。 舊版允許50個價格群組。
   * 輸入每個群組的&#x200B;**間隔值**&#x200B;或價格範圍。 最大值為40,000,000。
1. 按一下&#x200B;**[!UICONTROL Save]**。

   更新後的設定約需15分鐘才能在店面提供。

### 欄位說明

| 欄位 | 說明 |
|--- |--- |
| 選擇數量 | 指定可在店面中作為搜尋篩選器使用的價格範圍群組數目。 預設值： 8，最大值： 100 （截至[!DNL Live Search] 4.4.0） |
| 間隔值 | 指定每個群組的價格範圍間隔。 例如，間隔值為20的五個選取專案會建立0-20、20-40、40-60、60-80和>80的五個群組。 預設值：5，最大值：40,000,000 |

## 語言 {#language}

語言設定可告知[!DNL Live Search]讀取目錄和寫入索引時預期的語言。

語言有不同的語法規則集：例如，如何分隔字詞、動詞句子和字詞形式。語言設定可確保將正確的規則集套用到索引機制。

將語言設定設為目錄的主要語言。 變更索引的語言時，視目錄的大小和複雜度而定，可能需要5到60分鐘的時間才能在店面反映變更。
