# 建議型別

Adobe Commerce提供了一大組建議，您可以將這些建議部署至網站上的各個頁面。 所有建議型別都是資料導向。 它們由行為資料、產品屬性資料和量度提供支援。 為方便參考，建議型別分組如下：

- [個人化](#personalized)
- [交叉銷售和追加銷售](#crossup)
- [人氣](#popularity)
- [高效能](#highperf)

作為最佳實務，Adobe建議您在使用Recommendations時遵循下列准則：

- 讓您的建議型別多樣化。 如果客戶一再建議相同的產品，就會開始忽略建議。

- 請勿將相同的建議部署至購物車頁面和訂單確認頁面。 請考慮將`Most Added to Cart`用於購物車頁面，並將`Bought This, Bought That`用於訂單確認頁面。

- 保持網站整潔。 請勿在同一頁面上部署超過三個建議單位。

- 如果您的商店銷售服裝，`More like this`建議可能會建議與檢視產品性別不符的性別特定產品。 請考慮僅針對非服裝類別使用此建議型別。

>[!NOTE]
>
>如需本文所述事件的詳細資訊，請參閱開發人員檔案中的[storefront事件](https://developer.adobe.com/commerce/services/shared-services/storefront-events/#product-recommendations)。

## 資料需求和行為

Product Recommendations是資料導向系統，需仰賴從店面收集的行為資料。 建議的品質和數量取決於可用的事件資料量。

>[!IMPORTANT]
>
>大多數建議型別需要足夠的行為資料（例如產品檢視、加入購物車動作和購買）才能產生有意義的結果。 系統通常需要幾天活躍的購物者活動才能建置準確的推薦。 請參閱[整備指標](create.md#readiness-indicators)，瞭解網站流量如何協助填入各種建議型別。

### 資料不足時會發生什麼事

當沒有足夠的事件資料可產生建議時，系統可以：

- 傳回建議單位的空白結果。
- 觸發[備份建議](events.md#backup-recommendations)，例如在尚未提供個人化建議時顯示`Most viewed`產品。
- 顯示推薦單位中少於[已設定](create.md)的產品。

## 個人化 {#personalized}

這些建議型別會根據特定購物者在您網站上的行為記錄，來建議產品。 例如，如果購物者先前瀏覽過外套或在您的網站上購買過外套，這些建議基本上是從中斷的地方開始，然後推薦其他外套或類似產品。

>[!NOTE]
>
>個人化建議需要購物者具備既定的行為記錄。 沒有足夠互動歷史記錄的新訪客或購物者將會看到[備用建議](events.md#backup-recommendations)，例如「檢視次數最多」的產品，直到它們在您的網站上產生足夠的行為訊號為止。

| 型別 | 說明 |
|---|---|
| 為您推薦 | 根據每位購物者目前和先前的現場行為推薦產品。 根據購物者的瀏覽和購買記錄，顯示高度相關的建議。 此建議型別在多數購物者開始網站歷程的首頁上有效。 對於您網站上未產生任何訊號來個人化體驗的首次購物者，Adobe Commerce會根據「檢視次數最多」的建議型別顯示產品。 但是，當購物者開始與網站上的產品互動時，建議的產品會即時調整其行為。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別&#x200B;<br/><br/>**建議的標籤：**<br/> — 僅供您使用<br/> — 為您推薦<br/> — 靈感來自於您的購物趨勢 |
| 最近檢視的專案 | 根據瀏覽器歷史記錄，顯示購物者最近檢視的產品。 任何已刪除的產品都會由推薦單位移除。 如果沒有瀏覽器歷史記錄，或套用篩選規則時的歷史記錄不足，則不會顯示建議單位。 如果結果包含的產品少於設定的數量，建議單位只會顯示傳回的產品。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最近檢視的專案<br/> — 再次檢視 |

## 交叉銷售和追加銷售 {#crossup}

這些推薦型別是社交證實型的，可幫助購物者找到其他人喜歡的內容，或是產品導向型，可幫助他們找到其他類似的產品。 推薦的產品通常與所選產品互補。

>[!NOTE]
>
>「已檢視這個專案、已檢視那個專案」、「已檢視這個專案、已購買那個專案」以及「已購買這個專案、已購買那個專案」推薦型別不會使用簡單的發生次數量度，而是使用更複雜的協同篩選演演算法，尋找不會偏向於熱門產品的&#x200B;*有趣的相似專案*。 用於通知這些建議型別的資料是根據購物者的彙總行為，這些行為衍生自您網站上的多個工作階段。 此資料並非根據網站上單一工作階段中發生的購物者行為。 這些建議型別可協助購物者尋找鄰近的產品，這些產品可能並不明顯適合與目前檢視的產品配對。
>
>這些建議型別需要大量的跨產品互動資料，以識別有意義的關聯。 產品目錄多樣性有限或流量較低的商店，在足夠的行為模式出現之前，可能會看到較少的建議。

| 型別 | 說明 |
|---|---|
| 已檢視這個專案，已檢視那個專案 | 建議消費者以目前檢視的產品檢影片率不成比例的產品。<br/><br/>**使用位置：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 檢視過此產品的客戶也檢視了(PDP) |
| 已檢視此專案，但購買了其他專案 | 建議消費者在檢視目前產品後，多半會購買不合比例的產品。 此型別可協助引導購物者發現他們原本可能未注意到的產品。<br/><br/>**使用位置：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 檢視此最終產品的客戶已購買<br/> — 客戶最終已購買<br/> — 其他人檢視此產品後會購買什麼？ |
| 已購買此專案，已購買該專案 | 建議購物者經常以目前檢視的產品購買不成比例的產品。 此型別顯示高度相關的產品，購物者可透過彙總其他購物者使用目前產品購買的內容，將其新增至購物車。<br/><br/>**使用位置：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 取得您所需要的一切<br/> — 不要忘記這些<br/> — 經常一起購買 |
| 更多相關資訊 | 根據類似的中繼資料（例如名稱、說明、類別指派和屬性）推薦產品。 透過評估正在檢視之產品的屬性，此型別會推薦相同類別中的類似產品。 例如，如果購物者瀏覽瑜伽墊，建議使用裝置類別中的其他產品。 由於此建議型別無法區分性別，因此不建議服裝、時尚或其他性別特定垂直領域使用。<br/><br/>**使用位置：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 更多類似這樣的產品<br/> — 類似這樣 |
| [視覺相似度](#visualsim) | 建議產品外觀與正在檢視的產品相似。 如果影像和產品的視覺方面對於購物體驗很重要，則此建議型別最有用。 |

## 人氣 {#popularity}

這些建議型別會建議過去七天內最受歡迎或最熱門的產品。

>[!NOTE]
>
>基於人氣的建議需要店面提供足夠的事件資料。 如果您的存放區為新存放區或低流量，在收集到足夠的行為資料之前，這些建議型別可能會傳回有限的結果或無結果。 監視您的[資料整備程度指標](workspace.md)，以確保最佳效能。

| 型別 | 說明 |
|---|---|
| 檢視次數最多 | 建議檢視次數最多的產品，方法是計算過去七天內發生檢視動作的工作階段數。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最受歡迎<br/> — 趨勢<br/> — 目前最受歡迎<br/> — 最近最受歡迎<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 購買最多 | 建議購物者過去七天內最常購買的產品。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最受歡迎<br/> — 趨勢<br/> — 目前最受歡迎<br/> — 最近最受歡迎<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 加入購物車次數最多 | 建議購物者在過去七天內最常新增到購物車中的產品。 此建議型別可用於所有頁面。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最受歡迎<br/> — 趨勢<br/> — 目前最受歡迎<br/> — 最近最受歡迎<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 趨勢 | 根據產品在您網站上的人氣近期動向推薦產品。<br/><br/>Adobe Sensei會彙總您網站上的瀏覽與購買資料，以判斷與排名哪些產品最近受到購物者歡迎。 由於「趨勢」會分析最近的產品動向，因此對於營業額高的目錄來說，這是一種有效的建議型別。 如果您的目錄比較靜態，除非您對象的購物模式是高度變數，否則可能沒有用處。<br/><br/>當在首頁上使用時，Trending會推薦最近在整個網站都很受歡迎的產品。 趨勢分析顯示的不是持續受歡迎的產品，而是最近受歡迎的產品。 例如，如果您有促銷特定產品的電子郵件行銷活動，電子郵件產生的熱門程度增加會增加促銷產品歸類為趨勢的可能性。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 趨勢<br/> — 目前趨勢<br/> — 最近趨勢<br/> — 熱門產品<br/> — 趨勢相關產品(PDP) |

## 高效能 {#highperf}

這些建議型別會根據成功標準（例如加入購物車或轉換率），建議表現最好的產品。

>[!NOTE]
>
>高效能建議型別仰賴轉換資料（購買和加入購物車的動作）。 轉換量低的新商店或商店，可能需要在這些建議生效前，於7-14天內收集資料。

| 型別 | 說明 |
|---|---|
| 檢視購買轉換 | 推薦檢視與購買轉換率最高的產品。 在所有註冊了產品檢視的購物者工作階段中，最終註冊購物者購買的比例為何？<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最暢銷商品<br/> — 熱門產品<br/> — 您可能感興趣 |
| 檢視到購物車轉換 | 推薦最高檢視到購物車轉換率的產品。 在所有註冊了產品檢視的購物者工作階段中，購物者最終註冊加入購物車的比例為何？<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最暢銷商品<br/> — 熱門產品<br/> — 您可能感興趣 |
| 購買最多 | 此建議型別通常稱為「最暢銷商品」，會計算過去七天內發生下單動作的工作階段數量。 此建議型別可用於所有頁面。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最受歡迎<br/> — 趨勢<br/> — 目前最受歡迎<br/> — 最近最受歡迎<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 加入購物車次數最多 | 建議購物者在過去七天內最常新增到購物車中的產品。 此建議型別可用於所有頁面。<br/><br/>**使用位置：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最受歡迎<br/> — 趨勢<br/> — 目前最受歡迎<br/> — 最近最受歡迎<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |

## 視覺相似度 {#visualsim}

_視覺相似度_&#x200B;建議型別會推薦外觀與正在檢視之產品類似的產品。 當產品的影像和視覺方面是購物體驗的重要部分時，此建議型別最有用。

### 運作方式

_視覺相似度_&#x200B;建議型別針對目錄中與目前檢視的影像具有視覺相似度的其他產品提供建議。 視覺相似度包含下列方面：

- 顏色
- 形狀
- 大小
- 紋理
- 材質
- 樣式

Adobe Sensei使用AI處理和分析目錄中的影像，並建立用於判斷視覺相似性的屬性。

>[!NOTE]
>
> 如果您在非生產環境中測試此建議型別，請確定您的影像URL可公開存取。

>[!NOTE]
>
> 目前，產品影像的大小不得超過10 MB。

由於此建議型別不適用於大部分的目錄，因此預設不會啟用。 您必須明確啟用此建議型別。

### 啟用視覺相似度建議型別

>[!NOTE]
>
> 當您&#x200B;_安裝_&#x200B;視覺相似度[推薦型別作為選用模組時，即可使用](install-configure.md)視覺相似度推薦型別。

1. 在&#x200B;_管理員_&#x200B;側邊欄上，前往&#x200B;**行銷** > _促銷活動_ > **產品建議**&#x200B;以顯示&#x200B;_產品建議_&#x200B;儀表板。

1. 按一下「**設定**」（齒輪圖示）以顯示「_設定_」頁面。

1. 在&#x200B;_視覺建議_&#x200B;區段中，選取以&#x200B;**啟用視覺建議**。

1. 完成時，按一下&#x200B;**儲存變更**。

   當頁面型別為[產品詳細資料](create.md)時，**建立新建議**&#x200B;頁面現在會將&#x200B;**視覺相似度**&#x200B;顯示為可選建議型別。

啟用視覺化建議後，Adobe Sensei會啟動影像處理。 所需時間取決於目錄的大小。

### 使用處

- 產品詳細資料

### 建議的店面標籤

- 您可能也會喜歡
- 我們找到您可能喜歡的其他產品
- 受此款式啟發
