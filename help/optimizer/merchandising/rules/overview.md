# 銷售規則

銷售規則結合邏輯與動作，以塑造產品在&#x200B;**搜尋結果**、**預設產品清單** （**所有產品清單**）及&#x200B;**類別頁面** （[類別規則](#category-rules)為測試版）中的顯示方式。 您可以提升、隱藏、釘選或隱藏產品，並套用&#x200B;**智慧排名**，讓清單反映您的業務目標。

每個&#x200B;**搜尋規則**&#x200B;有三個主要元件：

- **條件** — 當購物者的搜尋符合時，會觸發動作的查詢式需求。
- **事件** — 符合條件時發生的動作（手動排名和相關事件）。
- **詳細資料** — 規則的名稱，以及選擇性的時間範圍和說明。

**類別規則**&#x200B;使用&#x200B;**類別選擇**，而非搜尋查詢條件；智慧型排名和手動排名的運作方式與搜尋相同，差異顯示在[建立和管理規則](add.md)中。

您可以結合搜尋規則的多個條件和動作，並排定任何規則在期間內為作用中。 您也可以設定&#x200B;**預設規則** （**所有產品清單**），此規則適用於不再套用特定搜尋或類別規則的情況。

## 類別規則 {#category-rules}

>[!IMPORTANT]
>
>類別規則為測試版。

**類別規則**&#x200B;控制&#x200B;**類別頁面**&#x200B;上的產品順序。 您可以選取一或多個類別，然後套用智慧型排名（例如檢視次數最多、趨勢分析等）和手動動作，例如釘選、提升和埋藏。 他們不使用搜尋查詢條件。 如需設定步驟、規則型別，以及類別與搜尋的排名套用方式，請參閱[建立和管理規則](add.md)。

## 需求

簡單&#x200B;**搜尋規則**&#x200B;可以有單一條件和單一事件，而複雜規則最多可以有十個條件來觸發最多25個事件。 **類別規則**&#x200B;遵循手動排名的相同事件限制；它們不使用查詢條件。

規則可以有：

- 最多10個&#x200B;**條件** （僅限搜尋規則）
- 最多25個&#x200B;**事件**

查詢文字可以包含：

- 英數字元（字母和數字）
- 大小寫字元。 將忽略大小寫。

## 邏輯運運算元

邏輯運運算元`AND`和`OR`結合兩個條件並傳回不同的結果。 在具有多個條件的規則中使用的所有邏輯運運算元都是相同的。 無法在相同規則中同時使用`AND`和`OR`。

### 比對運運算元

Match運運算元`All`和`Any`決定用來聯結規則中多個條件的邏輯運運算元，而且可用來變更現有的運運算元。

- `All` — 使用`AND`邏輯運運算元聯結多個條件。 使用`All` Match運運算元的規則只能有一個`Search query is`條件。
- `Any` — 使用`OR`邏輯運運算元聯結多個條件。

構成複雜規則時，可以協助您以縮排寫出規則，以描述傳回想要達到的結果所需的條件、關聯事件和產品名稱或SKU。 然後，建立規則並測試結果。

## 預設規則

您可以設定預設規則（**所有產品清單**），此規則會在未提供搜尋字詞或無法套用其他搜尋規則時套用。 如果您將預設規則設為「最多購買」，則查詢會預設為該排名型別，除非由更具體的搜尋字詞取代。 預設規則無法設定搜尋字詞。 **類別規則**&#x200B;是獨立的：它們只適用於您選取的類別，不會取代預設清單規則。

## 多個規則的優先順序

下列專案適用於&#x200B;**搜尋規則**，以及它們在指定搜尋中互動的方式。 **類別規則**&#x200B;適用於每個類別；請參閱[建立和管理規則](add.md#category-rules)，瞭解這些規則如何配合搜尋和預設規則。

一次只對一個搜尋字詞套用一個搜尋規則。
如果發現多個規則適用於搜尋片語，則會套用所有這些規則。 如果兩個規則（`rule 1`會增加SKU1，但`rule 2`會隱藏相同的SKU）之間發生衝突，則優先使用最近套用的規則(`rule 2`)。

- 規則會依「上次修改」時間戳記排序。 最近修改的規則會依時間戳記順序先套用，之後則套用較舊的規則。
- `query is`條件優先於其他條件。 如果較新的規則包含`query contains`條件，但較舊的規則有`query is`條件，則套用`query is`規則。

### 店面請求

如果包含`query is`條件的作用中規則符合搜尋字詞，則會套用該規則。 如果有多個具有`query is`條件的相符規則，則會套用最近更新的作用中規則。
否則，會套用最近更新的使用中規則。

### 預覽請求

在[!DNL Adobe Commerce Optimizer]中提出的要求運作方式略有不同。 預覽[!DNL Adobe Commerce Optimizer]時，會套用所有規則，包括已到期和已排程的規則。

- 如果正在預覽的規則具有`query is`條件，則會套用它。
- 如果正在預覽的規則沒有`query is`條件，但找到具有`query is`條件的後續作用中相符規則，則會套用`query is`規則。
- 如果正在預覽的規則沒有`query is`條件，而且找不到具有`query is`條件的其他規則，則會套用正在預覽的規則。
